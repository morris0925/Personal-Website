StreamLens

A real-time pipeline that ingests live GitHub events, serves them for analysis as they arrive, and diagnoses its own anomalies with an agentic RAG loop — while keeping a human in charge of the ticket.

The problem

Streams tell you what, not why

Most event pipelines stop at "the number moved." Someone still has to notice the anomaly, remember a similar incident, and write up a hypothesis by hand — usually after the fact, from a dashboard that only shows the symptom.

The approach

A diagnosis loop, with a human veto

When a metric like push rate deviates, the system retrieves similar past incidents from DuckDB's vector index and has an LLM rank hypotheses by confidence. Auto-ticketing stays deferred on purpose: it keeps a human reviewing the call, and keeps LLM cost and failure modes bounded.

Guardrail

Deferred by design

The tempting next step is to let the agent open the ticket itself. StreamLens holds that back deliberately — a ranked hypothesis is cheap to generate and easy to get wrong, so a human stays the one who acts on it.
